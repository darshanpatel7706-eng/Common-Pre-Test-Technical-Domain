# Section G: Technical Mindset & Problem Solving

## Q1. Learning a New Technical Skill
To learn a new technology, I:
- Start with basic concepts
- Practice regularly
- Build small projects
- Teach or explain concepts to others

This helps ensure deep understanding.

---

## Q2. When Stuck on a Technical Problem
If I am stuck for several hours, I:
- Break the problem into smaller parts
- Refer to documentation and online resources
- Ask for help from communities or mentors

---

## Q3. Explaining Technical Concepts to Non-Technical People
I use simple real-life examples, such as:
- A database as a digital cupboard
- An API as a waiter taking orders

This makes concepts easy to understand.

---

## Q4. Debugging Beyond Code Errors
Debugging is about finding the root cause of a problem.
It requires logical thinking, patience, and a systematic approach.

---

## Q5. Importance of Documentation
Documentation is important because:
- It helps future understanding
- Makes maintenance easier
- Improves collaboration
